The LSB UI

The LSB is a multiple window MDI (Multiple Document Interface) application. It starts with the main window, and then when user opens a new view file, the new view is contained in its own separate window. If user opens the same view file again, a new window will be created for the view file. Despite the view file is same or not, there is always one window for each view instance.

Main window

Here's how the main window looks like,

Menu

The File menu

  • Open view ... - This menu item launches the open view dialog for user to select a view file and open as LSB view.
  • Open recent view - Select a previously opened view file from the sub-menu and open as LSB view.
  • Close all views - This will close all the currently opened views.
  • Manage view history ... - This menu item launches the view file history management window to configure how the view files are ordered in the opened view history.
  • Exit - This exits and closes the LSB application.

The Windows menu

  • Views - This sub-menu contains entries for all currently opened LSB views. Selecting an entry in the sub-menu will bring the corresponding LSB view into front/make it active view.
  • Global log window - This menu item brings the global logging window into front. It logs the view open/close and all application level events.

The Option menu

  • Settings ... - This is the only menu item in this menu. It launches the LSB application settings dialog for user to change configurations for the LSB application.

The Help menu

  • Help ... - This shows the help content for the LSB application, including the content you are reading now.
  • About ... - This shows the about information for the LSB application.

Toolbar Buttons

In the main window, there is also a toolbar that contains a set of buttons for the functionalities. The funtionalities provided by these buttons are already available through the menu items in the main menu.

  • Open view - This button has the same functionality as the "Open view ..." menu item under File menu. Clicking the button will launch the open view dialog to select and open LSB view. Besides, the little triangle next to it triggers a drop down menu that shows the same view history as the Open recent view does.
  • Close all views - This button is same as the Close all views menu item under File menu, it closes all currently opened LSB views.
  • Manage history - This is another entry to the view file history management window.
  • View - This button will launch a Select viiew dialog for user to select a view from a view list which contains all currently opened LSB views and make it the active view. There is also a button with little triangle next to it which triggers drop down menu that contains menu items mapped to all currently opened views. Select a menu item here will bring the corresponding LSB view into front.
  • Log window - This button brings the global logging window into front.
  • Settings - This button launches the settings dialog for the LSB application.
  • Help page - This button shows the help window for the LSB application.

  • About LSB - This button shows the about window for the LSB application.

View window

Here's a typical view window, it consists of 4 tabs, View, Logs, Lua and About.

View

The View tab shows the content for the LSB view.

Logs

The Logs tab contains the local logging - all the events happened within the LSB view instance, including both parameter checking and parameter changes, etc.

Lua

The Lua tab contains the Lua code snippets executed within the LSB view instance.

About

The About tab contains the about information for the LSB view.

Open view dialog

The open view dialog is used to find, select and open a LSB view file in the LSB application. It consists of a view list and view preview pane. When a LSB view is selected in the view list, the corresponding information of the view will be displayed in the preview pane.

The view list also supports smart filter, see The Smart Filtering in ListView. This enables user to quickly locate the view through combination of column based filters. When a view is selected, user may also click on the Open the view folder button to open the target view folder using the OS native file management application, such as the File Explorer on Windows, Safari on Mac OS X, etc.

Manage opened view history

This is the dialog that manages how the view history is displayed.

  • History count settings
    • Maximum item count in menu - This configures how many items will be displayed in the view history menu.
    • Maximum item count in history file - This configures how many items will have the history data stored in the history data file.
  • How are the view files ordered
    • Most recently used time - The view files are ordered by most recently used time
    • How many times view used - The view files are ordered by how many times the view is used
  • View file history - This is a list of view files opened before. They are ordered based on setting configured above.
