The LSB UI

The LSB application uses a multi‑window, multiple‑document interface (MDI). At startup, only the main window is displayed, containing the menu bar and toolbar. When the user opens an LSB view file — whether from the menu or the toolbar — the view is loaded into its own independent window. Opening the same file again creates an additional window for that new instance. Each LSB view, regardless of whether it comes from the same file or a different one, always runs in its own dedicated window.

1. Main window

Here's the main window of the LSB application,

1.1 Menu

1.1.1 The File menu

  • Open view ... - Opens the “Open a view” dialog, allowing the user to choose a view file from the LSB view list and load it as an LSB view.
  • Open recent view - A sub‑menu containing entries for previously opened LSB views. Selecting an item reopens that specific view immediately.
  • Close all views - Closes all currently open LSB view windows.
  • Manage view history ... - Opens the “Manage opened view history” dialog, where the user can configure how previously opened view files are ordered within the “Open recent view” sub‑menu.
  • Exit - Closes all open LSB view windows and exits the LSB application.

1.1.2 The Windows menu

  • Views - A sub‑menu listing all currently open LSB views. Choosing an item brings that view’s window to the front and makes it the active view.
  • Global log window - A toggle that shows or hides the “Global logging” window. If the window is not currently visible, selecting this option displays it and brings it forward. If it is already visible, selecting the option hides it without closing it. The “Global logging” window records view open/close events as well as other application‑level activities.

1.1.3 The Option menu

  • Settings ... - This is currently the sole item in the “Option” menu. Selecting it opens the LSB “Settings” dialog, where the user can adjust all application‑level configurations, including General settings, Colors & Fonts configurations, Shells settings, and Views repository configurations.

1.1.4 The Help menu

  • Help ... - Opens the “LSB Help” window, which displays the help documentation for the LSB application, including the page currently being viewed.
  • License ... - Opens the License page, which describes the license used for the application, how business users can use the application for comercial purpose.
  • About ... - Opens the “About LSB” dialog, which provides information about the application and credits the SDK libraries and other resources used in its development.

1.2 Toolbar Buttons

The main window includes a toolbar that provides quick access to several key functions. All of these functions are also available through the main menu. On macOS, where the application menu appears in the system’s global menu bar, the toolbar becomes the only visible element in the application’s main window.

  • Open view – Performs the same action as the “Open view …” command in the File menu. Clicking it opens the “Open a view” dialog for selecting and loading an LSB view. The adjacent triangle opens a drop‑down list showing the same recent‑view history as the Open recent view sub‑menu; selecting an entry opens that view immediately.
  • Close all views – Matches the “Close all views” command in the File menu and closes all currently open LSB views.
  • View history – Another way to open the “Manage opened view history” dialog, where the ordering of items in the Open recent view sub‑menu can be configured.
  • Views – Enabled only when at least one LSB view is open. It opens the “Select view” dialog, allowing the user to choose from all open views and activate the selected one. The accompanying triangle button opens a drop‑down list of all open views; selecting one brings that view to the front.
  • Global logs – Toggles the visibility of the “Global logging” window. If the window is hidden, clicking the button shows it and brings it forward.
  • Settings – Opens the “Settings” dialog, where users can adjust the application’s configuration.
  • Help page – Displays the “LSB Help” window containing the application’s help documentation.
  • About LSB – Opens the “About LSB” dialog, which provides information about the application.

2. LSB View window

An LSB view window generally includes at least four pages, View, Logs, Lua and About.

2.1 View page

The View page displays content produced from the XML configuration defined within the LSB view file. A wide range of controls — including both input widgets and data‑presentation components — are arranged throughout the view using various layout managers that determines their structure and positioning. In the “File Explorer View” example below, the root layout is a horizontal SplitLayout: the left pane contains a FileTreeView that shows the folder hierachy, while the right pane contains a vertical SplitLayout. Within this vertical SplitLayout, a FileListView at the top lists all files located in the folder currently selected in the FileTreeView, and a TextView at the bottom displays the contents of whichever file is selected in the FileListView. This arrangement provides a complete, interactive file‑browsing experience within a single LSB view.

2.2 Logs page

The Logs page records all local logging activity, providing a detailed account of every event generated within the active LSB view instance. Each control within the view performs a condition check — by default an empty script that always evaluates to true — to decide whether its associated data source should be accessed or refreshed and whether the control’s displayed content should be updated accordingly. The outcomes of these condition checks are displayed in the Message list at the top of the page. When a control updates one of its output parameters and that update subsequently triggers a corresponding input parameter change in another control, both of these related events are captured in the Parameter change list shown at the bottom. This logging mechanism helps view creators debug their designs by confirming that output parameters are being emitted with the correct values and that those values are successfully delivered to the intended target controls through their input parameters.

2.3 Lua page

The Lua page displays the Lua code snippets that are executed as part of the active LSB view instance. This page is always present, even when the view does not define any explicit Lua scripts, because every LSB view relies on a set of built‑in Lua functions that operate behind the scenes. These internal Lua routines run throughout the entire lifecycle of the view, from the moment it is loaded to the point where parameter values are evaluated and checked. For example, the system automatically generates an table named lsb within the Lua runspace, allowing Lua code to access shared information such as the view’s file path, file name, and view name, etc. All condition checks for view controls are written exclusively in Lua. The Lua page itself is read‑only and presents the executed Lua code with full syntax highlighting for easier inspection and understanding.

2.4 About page

The About page presents descriptive information related to the specific LSB view. All content shown on this page is fully configurable within the LSB view file, allowing creators to define exactly what appears there. This information may be produced directly from data embedded in the view file itself, or it can instead be sourced from an external web page that is publicly accessible on the Internet.

2.5 Other pages

If the view author incorporates external shells such as PowerShell, Bash, or Zsh to implement functionalities within an LSB view, the system automatically adds additional pages — one for each shell type used. Each of these shell‑specific pages displays the executed shell scripts together with the full output they produced. Both the view creator and the end user can rely on these pages to review the exact commands that were run and examine their resulting output, helping verify that the view’s content is being refreshed and updated correctly based on the underlying shell logic.

3. Dialogs

The LSB application makes use of several different dialogs that are employed throughout the interface to present various types of data and to allow users to modify configuration settings for the application. These dialogs serve as dedicated interfaces for viewing information and adjusting options that influence how the application behaves.

3.1 Open a view dialog

The Open a view dialog allows users to locate, choose, and open an LSB view file within the LSB application. The dialog is composed of two main sections: a view list and a preview pane. It automatically scans all configured view repositories, discovers available LSB view files, and populates the view list accordingly. When the user highlights a specific LSB view in the list, detailed information about that view appears in the preview pane. The view list also supports the smart‑filter feature (see The Smart Filtering in ListView for details), which enables users to quickly narrow down the list by applying combinations of column‑based filters. After selecting a view, the user can additionally click the “Open the view folder” button to open the directory containing the view file using the operating system’s native file manager, such as File Explorer on Windows or Finder on macOS.

3.2 Manage opened view history

This dialog controls how the view‑opening history is presented in the “Open recent view” sub‑menu located under the “File” menu.

  • History count settings
    • Maximum item count in menu - This option determines how many entries appear in the “Open recent view” sub‑menu. The allowed range is from 1 to 9 inclusive, and the value must always remain smaller than the Maximum item count in history file setting.
    • Maximum item count in history file - This option specifies how many previously opened view entries are saved in the persistent history data file. The valid range is from 1 to 99 inclusive, and the value must be greater than the Maximum item count in menu setting.
  • How are the view files ordered
    • Most recently used time - When selected, the list of view files is sorted according to the most recent time each view was opened.
    • How many times view used - When selected, the list is sorted based on how frequently each view file has been opened.
  • View file history - This section displays the complete list of previously opened view files. The ordering of this list depends on the selected sorting option above, meaning it will either follow the most‑recently‑used order or the usage‑count order.

3.3 Select view

The dialog appears when the user clicks the View button in the main window’s toolbar. It presents a complete list of all LSB views that are currently open within the application. When the user selects one of the listed views and then confirms the selection by clicking the OK button, that view is immediately brought to the front of the interface and becomes the active, focused view.

3.4 Settings

The LSB Settings dialog is not described in this section; instead, its full explanation is provided in the following section titled “The LSB Settings.” That section offers a detailed overview of all configuration options available within the dialog.
