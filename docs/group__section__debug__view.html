How to Test/Debug a LSB View

When you are creating a new LSB view, you definitely want to test it before you release it to other users. Here are the steps on how you can test a LSB view locally through the LSB application.

  1. Create a folder for the view to be created/tested.
  2. Create or put the view to be tested in the folder.
  3. Launch LSB and open the settings dialog, create a new view repository points to this folder.
  4. Open the view to be tested in LSB, now you can see the view content.
  5. In the logs tab in the view window, you can find how the parameters are populated and how parameters are passed between different view controls in the LSB view.
  6. In the Lua tab in the view window, you can find all the Lua scripts executed within the view to implement the functionalities, such as parameter check, data generation, etc.
