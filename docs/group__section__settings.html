The LSB settings

LSB has a dedicated settings dialog to configure the various settings for the application. All the settings are categorized in 4 groups. Check the following sections to learn what they are and how they changes the the LSB application behaves.

General

This is the general setting category for LSB. The settings included in this category are,

  • Aggregation time window for multiple input parameter changes - This defines length of a time window in unit of milliseconds. For a LSB view control, it could have multiple input parameters, and there might multiple parameters have the values chaged at the same time (or in a very short time range). If this time window length is set to 0, that means every parameter value change will trigger data load and content refresh in the view control. This is not expected behavior in most cases. By setting this time window to a short length, then all the parameter value changes happened within this window will be recognized as one action and trigger one data load and content refresh only. Please be reminded that this time window is a moving window. Assuming this time window is set to D, and the first input parameter value gets changed at T0, and then a second parameter gets changed at T1, where T1 - T0 <= D, these two input parameter changes will be recognized as one action, and the actual data load/content refresh will be triggered at time T1 + D. In general, if there are n parameters have their values changed around same time (with time gap between each adjecent two parameters less than D), then the actual data load/content refresh will happen at Tn + D.
  • Default view size - This defines the default window size for LSB view. When a LSB view is open, it will be set to this size by default.
  • Show the GIT client window during execution for enlistment or refresh - The LSB application uses GIT to synchronize the default view repository and the help document repository. It calls the git command to get the files synced. This setting controls if the console window is shown during the synchronization.

Color & Fonts

This category contains the configurations for LSB UI, including colors and fonts, etc.

  • Active view caption - This configures the color and font for the active view controls. User may click on the foreground color, background color to chage the colors, and click on the "Choose..." button to select the font to be used for the active view control. The Preview control will show the final effect for any changes made on these configurations.
  • Inactive view caption - This configures the color and font for the inactive view controls. It can be changed and previewed in the same way as the active view caption does.
  • Shell window content - This is the setting for the shell window content. When a shell window is needed, such as the GIT console window mentioned above for the view synchronization, it will be displayed with the colors and font configured here. The settings can be changed and previewed in the same way as the active view caption and inactive view caption.

Shells

This is the category for configuring the shells to be used in LSB views. The page is composed of a shell list and a set of tool buttons that manages the shell list.

Here are what the tool buttons do,

  • Scan - This scans all known shells supported by LSB on current Operating System. The found shells will be added into the shell list automatically with default settings.
  • Edit - This button is enabled only if an item in the shell list is selected. Clicking on the button will trigger the shell setting editor dialog to change the settings for the selected shell. See section below for the details of shell setting editor dialog.
  • Add - This adds a new shell into the shell list. It also launches the shell setting editor dialog for configuring the settings for the new shell.
  • Delete - This deletes the currently selected shell from the shell list. It's enabled only if a shell item is selected in the shell list.
  • Clear - This removes all shells from the shell list.

Note: Double click on an item in the shell list has the same effect as clicking the Edit button.

Shell Settings Editor

When user requests to add a new shell or edit an existing shell, following editor dialog will be launched for user to edit the settings,

Here are the settings user can set for a shell,

  • Name - Name of the shell
  • Path - The path to the shell executable
  • Reference / Module list - This is a list of modules to be referenced in the shell. The modules listed here will be automatically loaded when the shell is launched.
  • Placeholders - These placeholders will be used to embed certain values into the wrapper script that helps to execute the user specified script through the shell process.
    • Customized user script - This is the placeholder for the user script to be executed.
    • Intermediate data file - This is the placeholder for the full path of the intermediate data file. The LSB view control will try to load the generated data from this intermediate data file if it is used in the wrapper script.
    • First half of signature - To detect the wrapper script execution is completed, the LSB uses two parts of signature, combine them together and output the complete signature at the end of script execution. When LSB finds the full signature in the shell console output, it knows that the script execution completes. This placeholder is for the first half of the signature data. The signature is broken into two parts so the assignment of the data won't mistakenly trigger the recognization of execution complete.
    • Second half of signature - This is the placeholder for the second half of signature data.
    • Path to view file - This is the placeholder for the full path to the currently open view file.
  • User script wrapper - This is the script that wraps the actual user script to be executed in the shell. In the wrapper script, it can help to output data into the intermediate data file in specific format, it can help to generate the signature data and output to shell console to enable the determination of script completion, or any other helper function needed for the user script execution.
  • Script execution timeout - This is the default timeout setting for the user script execution. Each user script could have its own customized timeout set through the script data source configuration, see ScriptData.

Views

This is the category for configuring the view repositories to be used in LSB. Similiar to the Shells, this page is composed of a view repository list and a set of tool buttons that manages the view repository list.

Here are what the tool buttons do,

  • Edit - This button is enabled only if a view repository is selected in the list. Clicking on the button will open the view repository editor dialog to change the settings for the LSB view repository. See section below for the details of view repository setting editor dialog.
  • Synchronize - If the currently selected view repository is a GIT repository and the remote GIT repository is provided, this button is enabled. Clicking on the button will trigger the synchronization of the view files (from the remote GIT repository to local copy).
  • Add - This adds a new view repository into the repository list. It also launches the view repository setting dialog for configuring the settings for the view repository.
  • Delete - This deletes the currently selected view repository from the repository list. The button is enabled only if a view repository item is selected in the view repository list.
  • Clear - This removed all view repositories from the repository list.

View Repository Editor

When user requests to add a new LSB view repository or edit an existing view repository, following editor dialog is open for editing the settings.

Settings can be changed in this dialog for a view repository,

  • Name - Name of the LSB view repository
  • Local view root - This is the local path to the root folder of the view repository.
  • GIT repository URL - If a view repository is synchronized from a remote GIT repository, this is the GIT repository URL. If it's empty, then it means this view repository is a local repository, no synchronization is needed for it.
  • Refresh interval - This is only valid if the view repository is synchronized from a remote GIT repository. And it controls how frequently that the LSB should do synchronization with the remote GIT repository. Please be reminded that this automatic synchronization only happens when LSB is launched. The other thing to mention is that the manual synchronization by clicking on the Synchronize button in the Views configuration page does not honor this interval setting, it does synchronization immediately when the button is clicked.
  • The repository is enabled to use the views hosted in it - This is a switch that controls if the LSB views in this view repository will be visible in the open view dialog.
