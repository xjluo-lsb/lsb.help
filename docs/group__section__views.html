The LSB View Repositories

The LSB views are managed through view repositories, either locally or remotely. In each of view repositories, the views can be futher categorized through folders (and sub-folders). The LSB will recursively find all the view files under the root folder of a view repository and show them in the Open View dialog. There could be multiple views with the same name in the same view repository, as long as they are put in different folders.

Default View Repository

The LSB applciation comes with a default view repository that is available at https://github.com/xjluo-lsb/lsb.views. When the LSB is launched, if the default view repository is not available, LSB will automatically clone the repository and make the views available to user. The view repository will be named as DefaultViews. This view repository is automatically configured and it's invisible in the view repository configuration dialog.

User Configured View Repositories

Users can configure their own repositories for use in LSB. As described in the View Repository Editor, users can configure a local-only repository by pointing the local view root to a local folder without providing a GIT repository URL. In this case, all views from this repository are available locally and not shared with others.

Users can also configure a remote view repository by providing both the GIT repository URL and the local view root folder. In this case, all LSB views in the GIT repository will be automatically synced to the local folder each time the LSB application is launched. This allows users to easily reuse LSB views created by others or share their own views with others.

For the community version, users can use the default view repository that syncs from a repository hosted on GitHub. Additionally, users can add one local repository for creating and managing LSB views. However, adding remote repositories or more than one local repository is not supported. For the paid version, there are no limitations. Users can add as many remote repositories as they want, and all these repositories will be automatically synced during the launch of LSB. Users can also add as many local repositories as they want, although this might not be a strong need.

In the view repository setting, user may also temporarily enable/disable a view repository by check/uncheck "The repository is enabled to use the views hosted in it". When there are too many views available through multiple view repository, this might be helpful to shorten the view list in the Open View dialog.

Sample Views

In the DefaultViews view repository, the LSB application already provides a set of sample views under the samples folder. These views are used to demonstrate what view controls are available in LSB, what a LSB view can do, how parameters are transmitted around, etc. They are good examples for learning how to create your down LSB view.
