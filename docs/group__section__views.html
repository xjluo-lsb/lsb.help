The LSB View Repositories

LSB views are organized and accessed through view repositories, which may reside either locally on the user’s machine or remotely in a Git-based source. Within each repository, views can be further organized into folders and sub‑folders to create a structured hierarchy. When the LSB application loads the available views, it recursively scans the entire directory tree under the repository’s root folder and displays all discovered view files in the view list shown in the “Open a view” dialog.

Multiple views may share the same file name within a single repository; as long as they are located in different folders, they can coexist without conflict.

Default View Repository

The LSB application includes a built‑in default view repository, which is a Git repository hosted at:

https://github.com/xjluo-lsb/lsb.views

When the LSB application is launched, it checks whether this default repository is already available locally. If it is missing, the application automatically clones the Git repository so that the default views become immediately accessible to the user.

This repository is internally named DefaultViews, and its configuration is managed entirely by the application. Because it is a core component of the system, it cannot be modified or removed through the repository‑configuration page in the Settings dialog.

User Configured View Repositories

Users may define additional view repositories to extend or customize their LSB environment.

Local‑Only Repositories

A user can configure a repository that exists solely on the local machine by specifying a local folder as the Local view root and leaving the Git repository URL field empty. In this configuration, all views are stored and accessed locally, and no synchronization with a remote source occurs. These views remain private to the user unless shared manually.

Remote Git‑Backed Repositories

Users may also configure a repository that synchronizes with a remote Git source by providing both a Git repository URL and a corresponding Local view root folder. Whenever the LSB application starts, it automatically synchronizes the local folder with the remote Git repository, ensuring that all LSB views are up to date. This setup allows users to reuse views created by others or distribute their own views across a team or organization.

Temporarily Enabling or Disabling Repositories

Within the repository settings, users may toggle the option “The repository is enabled to use the views hosted in it.” Disabling a repository hides all of its views from the “Open a view” dialog without deleting the repository or its files. This feature is particularly useful when many repositories are configured and the view list becomes too long; temporarily disabling some repositories helps reduce clutter and streamline navigation.

Sample Views

In the DefaultViews repository, the LSB application includes a collection of sample views located under the samples and common folders. These sample views illustrate the range of view controls available in LSB, demonstrate how an LSB view behaves in practice, and show how parameter values can be passed, transformed, and consumed across different components. They serve as practical, hands‑on examples for users who want to learn how to design and build their own LSB views.

Below are several representative examples drawn from these sample folders:

all-controls-demo.lsbv

This view demostrates majority of the controls supported in the LSB. It requires pwsh (PowerShell or PowerShell 7+) to be configured to run the scripts and demostrates following features,

  • Using Lua scripts defined in script reference, using PowerShell scripts defined in script reference.
  • Using PowerShell script for generating setting data for the PropertyInput control, which overrides the property definitions configured in the view file.
  • String input in the PropertyInput is sent to the ListView control for updating the caption for the ListView.
  • Demonstrates the context menu in the ListView.
  • Uses external Web site page (the wxWidgets home page) as the about page.
  • Data controls used in the view: ListView / TextView / TreeView / ChartView / TreeListView / WebView
  • Input controls used in the view: TextInput / ComboBox / PropertyInput / ButtonGroup
  • Layout controls used in the view: SplitLayout / TableLayout / TabLayout

file-explorer.lsbv

This view illustrates the FileTreeView and FileListView controls provided in LSB to compose a view with functionalities similar to the file managers (such as Windows Explorer on Windows, Finder on macOS, GNOME Files on Fedora, etc).

Besides, this view also demonstrates the context menu configuration on the FileListView control and how to call into function lsb.showMessage() in the Lua script to show message from the context menu. Double click on a plain text files listed in the FileListView control will trigger the view to show the content in the TextView control.

This view uses Lua scripts only for the functionality implementation, thus it supports all operating systems.

Graphs and Charts

There are 3 graph and chart sample views available, using different frameworks to draw the graphs and charts.

  • chartjs-sample.lsbv - This view uses the Chart.js library inside WebView control to visualize data. Credit to the contributors of the Chart.js library. Please visit https://www.chartjs.org/ to find more details about the library.
  • d3-sample.lsbv - This is another view uses the WebView control to render the graphs, while it uses the well-known D3 library (https://d3js.org/) to visualize the data. It contains three examples copied and modified from the original D3 samples. The Scatterplot is based on https://observablehq.com/@d3/scatterplot-tour, the Disjoint force-directed graph is based on https://observablehq.com/@d3/disjoint-force-directed-graph/2 and the Zoomable circle packing comes from https://observablehq.com/@d3/zoomable-circle-packing.
  • chartview-sample.lsbv - This sample view mainly demonstrates the ChartView in the LSB. It shows examples for all the chart types supported by ChartView, including Bar chart, Stacked bar chart, Column chart, Stacked column chart, Candlestick chart, Box plot, OHLC chart, Histogram, Pie chart, Polar area chart, Doughnut chart, Bubble chart, Scatter plot, Line chart, Math2D plot and Area chart. The data used in these charts are generated from Lua script that is defined in the script reference in the view. The ChartView control is built on top the wxCharts library. Credit to the authors of the wxCharts library, see more details at https://github.com/wxIshiko/wxCharts.

hexview.lsbv

This view illustrates how the HexView control provided by LSB can be used to display data or file in the hexadecimal form. It can open any file to be displayed in hexadecimal. This view demonstrates two different ways to load the file content into the HexView control,

  • Load file in Lua code, convert the content into hexadecimal string and send to the HexView control. This way can be used for displaying any data as long as the data is preprocessed to be hexadecimal string.
  • Send the file path to the HexView control and load the content directly in HexView.

A substantial collection of example views is already included in the default view repository, and this library will continue to grow over time as additional views are added. Users are encouraged to explore and experiment with these examples to understand the capabilities of LSB views, observe how different controls behave, and study how parameters are passed and processed. These sample views serve not only as demonstrations of what LSB can do, but also as practical reference material for learning how to design and build your own custom LSB views.
